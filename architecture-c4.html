Level 4-ish

Main Runtime Flow

For product discussion, a flow diagram is more useful than a code-class diagram here.

flowchart LR
    A[1. Visitor arrives] --> B[2. Create account or log in]
    B --> C[3. Trial wallet is provisioned]
    C --> D{4. Trial or paid balance available?}
    D -- Yes --> E[5. User enters full chat]
    E --> F[6. Backend calls LLM provider]
    F --> G[7. Usage is recorded in Firebase]
    G --> H[8. Wallet spend is updated]
    H --> I[9. Admin sees usage, wallet, payments, and users]
    D -- No --> J[5. User lands in wallet dashboard]
    J --> K[6. Checkout via Mercado Pago]
    K --> L[7. Webhook confirms payment]
    L --> M[8. Paid wallet becomes available]
    M --> E
