Transaction Path

  1. 01. Preference issued

    The page requests a MercadoPago preference from the backend and records the external reference in local storage.

  2. 02. Payment completed

    MercadoPago handles the transaction and returns the buyer to the success flow with the stored reference intact.

  3. 03. Key delivered

    The confirmation screen polls Firebase for approval and reveals the generated ChutesAI API key when it is available.

Operational note

When the frontend is served from the same origin as the Vercel functions, the page uses that origin automatically. If it is served elsewhere, set data-api-base on the body to point at the deployed backend.
